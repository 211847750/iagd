Update v1.0.4.0 of Grim Dawn broke some functionality

Close Zantai/Crate has announced that they are working on a fix for Grim Dawn.
Meanwhile, some extra work is required by you to prevent items being lost or duplicated.

Preventing item loss:


IA no longer allows you to transfer items while the game is running
You will need to close Grim Dawn in order to transfer items. (this is temporary)

An experimental work-around (~95% safe, MAY lose items) is to enable "instatransfer" under settings.
Instatransfer lets you transfer items into GD while the stash is open.

Enabling/disabling instaloot or instatransfer requires both Grim Dawn and IA to be closed before changes takes effect.


Obs: All items appear in tab 3 with Insta-Transfer, regardless of setting.

Preventing item duplication:


If you are playing Grim Dawn without mods: (like grimarillion etc)
  1. Go to settings
  2. Ensure that instaloot is NOT disabled
  3. Close Grim Dawn
  4. Close Item Assistant
  5. You can now loot items normally in Grim dawn without any issues


If you are playing Grim Dawn with mods: (like grimarillion etc)
  1. Place items in your stash
  2. Run away
  3. IA loots the items
  4. Close Grim Dawn
  5. Items have been looted without being duplicated

Close This is usually pretty easily resolved, by following this exact sequence.
Do not deviate from it, do it exactly, and it should resolve just about any problems you run into.
  1. Start Grim Dawn
  2. Start Item Assistant
  3. Close Grim Dawn
  4. Restart Item Assistant
  5. Go to mods/database
  6. Select vanilla
  7. Click Load database
  8. Click parse item stats
For more information see http://www.grimdawn.com/forums/showthread.php?t=35240&page=120

If you have just moved from GOG to Steam (or the other way around), IA may be detecting the wrong folder.
Removing (or temporarily moving) the old install should do the trick.
